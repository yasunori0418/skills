# ホワイトボックステスト技法（構造ベース）+ 経験ベース技法

コード構造から網羅を導く技法（ホワイトボックス）と、テスターの知識・直感に依る技法
（経験ベース）。各技法は「適する対象・手順・カバレッジ基準」を簡潔に示す。

## ホワイトボックス技法（構造ベース）

### ステートメントカバレッジ（statement coverage）

- **適する対象**: 実行可能なコードがある対象。最低限の構造網羅を保証したいとき。
- **手順**: すべての実行文を最低 1 回通るテストケースを設計する。
- **カバレッジ基準**: 実行文網羅率 = 実行した文 / 全文。100% でも全分岐は保証されない。

### ブランチ/デシジョンカバレッジ（branch / decision coverage）

- **適する対象**: 条件分岐（if/switch/ループ）を持つコード。ステートメントより強い網羅。
- **手順**: すべての分岐の真・偽（各エッジ）を最低 1 回通るケースを設計する。
- **カバレッジ基準**: 分岐網羅率 = 通った分岐エッジ / 全分岐エッジ。100% はステートメント
  100% を包含する。より厳密には条件網羅・MC/DC へ拡張する。

## 経験ベース技法

### エラー推測（error guessing）

- **適する対象**: 過去の欠陥傾向・よくある間違いが効く対象（境界・null・空・桁あふれ等）。
- **手順**: 経験・欠陥リスト・チェックリストから、壊れやすい入力・状況を推測してケース化する。
- **カバレッジ基準**: 形式的な網羅基準は持たない。想定欠陥リストの消化を目安にする。

### 探索的テスト（exploratory testing）

- **適する対象**: 仕様が薄い/流動的、短時間でリスクを洗いたい対象。学習しながら試す。
- **手順**: テストチャーター（探索の目的・範囲・時間）を決め、設計・実行・学習を同時に回す
  （セッションベースで記録を残す）。
- **カバレッジ基準**: 形式基準は持たない。チャーターの達成と発見欠陥・気づきの記録で評価。

### チェックリストベーステスト（checklist-based testing）

- **適する対象**: 定型の観点（セキュリティ・アクセシビリティ・入力検証等）を反復確認したい対象。
- **手順**: 観点チェックリストを用意し、各項目を対象に当てて確認する。
- **カバレッジ基準**: チェックリスト項目の消化率。
